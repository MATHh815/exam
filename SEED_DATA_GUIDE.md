#### 方法一: 直接运行脚本
```bash

```

```bash

```

## 注意事项

### 1. 数据重复
- 如果多次运行脚本,会生成重复的题目和试卷
- 建议在运行前清空数据库或删���旧数据

### 2. 管理员账户
- 如果数据库中已存在 `admin` 用户,脚本会跳过创建
- 如果需要重置管理员密码,请手动删除后重新运行

### 3. 题目ID
- 题目和试卷的ID是自动生成的
- 不要依赖固定的ID值

### 4. 数据库连接
- 确保 `.env` 文件中的数据库配置正确
- 确保数据库服务正在运行

## 清空数据

如果需要清空所有测试数据并重新生成:

```bash
cd exam/backend

# 删除数据库文件(SQLite)
rm instance/exam.db

# 重新初始化数据库
python init_db.py

# 重新生成测试数据
python seed_exam_data.py
```

## 自定义数据

如果需要自定义生成的数据,可以修改 `seed_exam_data.py` 中的:

### 1. 题目模板
修改 `QUESTION_TEMPLATES` 字典,添加或修改题目内容

### 2. 题目数量
修改 `generate_questions()` 函数的 `count_per_type` 参数

### 3. 试卷配置
修改 `create_exam_paper()` 函数调用时的参数:
- `name`: 试卷名称
- `duration`: 考试时长
- `description`: 试卷描述

### 4. 分值设置
修改 `create_exam_paper()` 函数中的 `score_map` 字典

## 常见问题

### Q: 运行脚本时提示模块找不到
A: 确保在 `exam/backend` 目录下运行,并且已安装所有依赖

### Q: 数据库连接失败
A: 检查 `.env` 文件中的数据库配置,确保数据库服务正在运行

### Q: 生成的题目数量不对
A: 检查 `generate_questions()` 函数的参数,确保每种题型的数量设置正确

### Q: 试卷没有显示在前端
A: 确保试卷的 `is_published` 字段为 `True`,只有已发布的试卷才会显示

### Q: 如何添加更多题目类型
A: 在 `QUESTION_TEMPLATES` 中添加新的题型,如 `fill_blank`(填空题)或 `essay`(简答题)
