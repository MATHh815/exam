# 学习计划功能完整修复

## 问题总结

创建学习计划时出现多个错误：
1. ❌ "缺少必填字段" - 前端使用 `target_date`，后端期望 `start_date` 和 `end_date`
2. ❌ "目标值用日期格式错误缺失" - 目标缺少 `period_start` 和 `period_end` 字段
3. ❌ "创建计划失败" - 考试类型值不匹配（前端用中文，后端期望英文）

---

## 修复内容

### 修复 1: 日期字段 ✓

**问题**: 前端只有 `target_date`，后端需要 `start_date` 和 `end_date`

**解决方案**: 
- 将表单的"目标日期"改为"开始日期"和"结束日期"
- 添加日期验证（结束日期不能早于开始日期）

```vue
<!-- 修改后 -->
<el-form-item label="开始日期" prop="start_date">
  <el-date-picker v-model="formData.start_date" />
</el-form-item>

<el-form-item label="结束日期" prop="end_date">
  <el-date-picker v-model="formData.end_date" />
</el-form-item>
```

### 修复 2: 目标周期字段 ✓

**问题**: 后端的 `_create_goal` 方法要求每个目标必须有 `period_start` 和 `period_end`

**解决方案**: 
在提交表单时，自动为每个目标添加这些字段（使用计划的开始和结束日期）

```javascript
const handleSubmit = async () => {
  // ...
  
  // 为每个目标添加 period_start 和 period_end
  const goalsWithPeriod = formData.goals.map(goal => ({
    ...goal,
    period_start: startDate,
    period_end: endDate
  }))
  
  const submitData = {
    ...formData,
    start_date: startDate,
    end_date: endDate,
    goals: goalsWithPeriod
  }
  
  emit('submit', submitData)
}
```

### 修复 3: 考试类型值 ✓

**问题**: 前端使用中文值（"公务员"、"考研"、"事业编"），后端期望英文值

**后端期望的值**（来自 `exam/backend/app/schemas/question_schema.py`）:
- `'civil_service'` - 公务员考试
- `'postgraduate'` - 考研
- `'public_institution'` - 事业编

**解决方案**: 修改表单选项的 value

```vue
<!-- 修改前 -->
<el-option label="公务员考试" value="公务员" />
<el-option label="考研" value="考研" />
<el-option label="事业编" value="事业编" />

<!-- 修改后 -->
<el-option label="公务员考试" value="civil_service" />
<el-option label="考研" value="postgraduate" />
<el-option label="事业编" value="public_institution" />
```

## 后端 API 要求

根据 `exam/backend/app/services/study_plan_service.py`：

### 必填字段
```python
required_fields = ['name', 'exam_type', 'start_date', 'end_date']
```

### 完整请求格式
```json
{
  "name": "全文",
  "exam_type": "civil_service",  // 必须是: civil_service, postgraduate, public_institution
  "start_date": "2025-12-26",    // 格式: YYYY-MM-DD
  "end_date": "2025-12-31",      // 格式: YYYY-MM-DD
  "description": "加油",         // 可选
  "goals": [
    {
      "goal_type": "exam_count",      // 必填
      "target_value": 10,             // 必填，正整数
      "period_start": "2025-12-26",   // 必填，格式: YYYY-MM-DD
      "period_end": "2025-12-31"      // 必填，格式: YYYY-MM-DD
    },
    {
      "goal_type": "daily_practice",
      "target_value": 10,
      "period_start": "2025-12-26",
      "period_end": "2025-12-31"
    }
  ]
}
```

### 目标类型 (goal_type)
- `'daily_practice'` - 每日练习题数 (1-500)
- `'weekly_practice'` - 每周练习题数 (1-500)
- `'daily_duration'` - 每日学习时长（分钟）(5-720)
- `'exam_count'` - 考试次数

### 目标值验证
```python

if goal_type in ['daily_practice', 'weekly_practice']:

# 学习时长
elif goal_type == 'daily_duration':
    # 必须在 5-720 分钟之间
```

## 常见错误和解决方案

### 错误 1: "缺少必填字段"
**原因**: `name`, `exam_type`, `start_date`, `end_date` 中有字段为空  
**解决**: 确保所有必填字段都已填写

### 错误 2: "日期格式错误"
**原因**: 日期不是 `YYYY-MM-DD` 格式  
**解决**: 前端已自动格式化，如果仍出错，检查日期选择器

### 错误 3: "结束日期不能早于开始日期"
**原因**: 选择的结束日期早于开始日期  
**解决**: 重新选择日期，前端已添加验证

### 错误 4: "无效的目标类型"
**原因**: `goal_type` 不在允许的值中  
**解决**: 使用下拉选择，不要手动输入

### 错误 5: "目标值必须是正整数"
**原因**: `target_value` 不是正整数  
**解决**: 使用数字输入框，不要输入负数或 0

### 错误 6: "目标周期日期格式错误或缺失"
**原因**: 目标缺少 `period_start` 或 `period_end`  
**解决**: 前端已自动添加，如果仍出错，检查代码
