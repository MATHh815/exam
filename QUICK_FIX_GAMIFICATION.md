## ❓ 常见问题

### Q1: 后端服务器无法启动

**错误**: `Address already in use`

**原因**: 端口 5000 被占用

**解决**:
```bash
# 查找占用端口的进程
netstat -ano | findstr :5000

# 杀死进程（替换 <PID> 为实际进程ID）
taskkill /PID <PID> /F

# 重新启动
python run.py
```

### Q2: 运行脚本时报错 "No module named 'app'"

**解决**:
```bash
cd exam\backend
set PYTHONPATH=%CD%
python init_user_points.py
```

### Q3: 数据库表不存在

**错误**: `no such table: user_points`

**解决**:
```bash
cd exam\backend
python migrate_phase1.py
python init_achievements.py
python init_user_points.py
```

### Q4: 刷新后仍然报错

**可能原因**:
1. 后端服务器未重启
2. 浏览器缓存问题

**解决**:
1. 重启后端服务器（Ctrl+C 然后重新运行 `python run.py`）
2. 清除浏览器缓存（Ctrl+Shift+Delete）
3. 强制刷新（Ctrl+Shift+R）

## 🔧 技术细节

### 错误原因分析

"基于了未知的路径格式" 错误通常由以下原因引起：

1. **后端服务器未运行**
   - 前端无法连接到 http://localhost:5000
   - API 请求失败

2. **用户积分记录未创建**
   - 数据库中没有 user_points 记录
   - API 返回错误

3. **数据库表未创建**
   - 未运行数据库迁移
   - user_points 表不存在

### 自动修复机制

系统具有自动修复功能：
- `PointsService.get_or_create_user_points()` 会自动创建积分记录
- 首次访问时会自动初始化

但如果后端未运行，自动修复无法工作。

### 手动初始化

`init_user_points.py` 脚本会：
1. 查询所有用户
2. 检查是否有积分记录
3. 为没有记录的用户创建初始记录
4. 提交到数据库
