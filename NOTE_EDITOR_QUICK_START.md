# 笔记编辑器增强 - 快速测试指南

## 快速测试步骤

### 1. 启动应用

```bash
# 启动后端
cd exam/backend
python run.py

# 启动前端（新终端）
cd exam/frontend
npm run dev
```

### 2. 测试新功能

#### 测试1: 创建新笔记并链接题目

1. 登录系统
2. 进入"笔记"页面
3. 点击"新建笔记"
4. 在编辑器中点击工具栏的"链接题目"按钮
5. 在搜索框输入关键词（如"Python"）
6. **观察搜索结果** - 应该只显示题目标题，不显示ID
7. 点击任意题目
8. **观察插入的文本** - 应该是 `[[题:题目标题...]]` 格式
9. 切换到"预览"模式
10. **观察渲染效果** - 应该显示为漂亮的链接：📝 题目标题...

#### 测试2: 验证向后兼容性

如果你有旧的笔记包含 `[[Q:123]]` 格式：
1. 打开旧笔记
2. 切换到预览模式
3. **观察旧链接** - 应该仍然正常显示为 "📝 题目 #123"

#### 测试3: 测试链接样式

1. 在预览模式下
2. 将鼠标悬停在题目链接上
3. **观察效果**:
   - 背景变为紫色渐变
   - 文字变为白色
   - 链接向上移动并放大
   - 出现阴影效果

#### 测试4: 测试长标题

1. 搜索一个内容很长的题目
2. 点击插入链接
3. **观察标题** - 应该自动截断为30字符并添加"..."

#### 测试5: 测试特殊字符

1. 搜索包含特殊字符的题目（如 `<script>` 或 `&nbsp;`）
2. 插入链接
3. 切换到预览模式
4. **观察渲染** - 特殊字符应该被正确转义，不会破坏页面

## 预期效果对比

**之前:**
```
题目 #123

这道题很重要：[[Q:123]]
```

这道题很重要：[[题:下列关于Python的描述正确的是...]]

渲染为: 这道题很重要：📝 下列关于Python的描述正确的是...

- 阴影效果
- 悬停时的动画（向上移动、放大、变色）

## 常见问题

### Q1: 旧笔记中的 [[Q:123]] 链接还能用吗？
A: 可以！系统完全向后兼容，旧格式链接仍然正常显示。

### Q2: 题目标题太长怎么办？
A: 系统会自动截断为30字符并添加"..."，确保链接不会太长。

### Q3: 如何知道链接指向哪个题目？
A: 链接文本就是题目的标题/内容，比ID更直观。未来可以添加点击跳转功能。

### Q4: 可以混合使用新旧格式吗？
A: 可以！同一笔记中可以同时使用 `[[Q:123]]` 和 `[[题:标题]]` 两种格式。

### Q5: 搜索对话框为什么变宽了？
A: 为了更好地显示题目标题和预览内容，提升阅读体验。
