- **掌握度计算**: 正确率(50%) + 练习次数(30%) + 最近表现(20%)

- **重要程度**: 被依赖次数 × 2 + 题目数量 / 10

## 🆕 最新更新 (2025-12-29)

### Feature: 学习计划功能改进 ✅

#### 改进概述
解决了用户反馈的三个核心问题：
1. 状态逻辑错误 - 计划创建后立即显示为"已完成"
2. 目标类型有限 - 只有2-3种基础目标类型
3. 缺少进度追踪 - 没有根据实际学习活动自动更新进度

#### 改进内容

**1. 智能状态管理** ✅
- 实现自动状态判断逻辑
- 根据日期和目标完成情况动态更新
- 状态规则：
  - 当前日期 > 结束日期 或 所有目标完成 → `completed`
  - 开始日期 <= 当前日期 <= 结束日期 → `active`
  - 用户手动暂停 → `paused`（不自动改变）

**2. 扩展目标类型** ✅
- 从 3 种扩展到 14 种（增加 367%）
- 支持 6 种科目的特定目标

**目标类型清单**：
- **练习数量类**（5种）：daily_practice, weekly_practice, monthly_practice, subject_daily_practice, subject_weekly_practice
- **学习时长类**（3种）：daily_duration, weekly_duration, subject_daily_duration
- **正确率类**（2种）：accuracy_rate, subject_accuracy_rate
- **考试类**（2种）：exam_count, exam_score
- **章节类**（2种）：chapter_completion, subject_chapter_completion

**3. 数据库改进** ✅
- 添加 `subject` 字段（VARCHAR(50)）- 支持科目特定目标
- 添加 `completed_at` 字段（DATETIME）- 记录完成时间

**4. 前端表单优化** ✅
- 目标类型按类别分组显示
- 科目特定目标自动显示科目选择器
- 根据目标类型动态调整输入范围和标签
- 智能默认值设置

#### 技术实现

**后端文件**（已修改）：
- `app/models/study_plan.py` - 添加新字段
- `app/services/study_plan_service.py` - 扩展目标类型，添加状态自动更新逻辑
- `app/routes/study_plans.py` - 无需修改，API完全兼容

## 🆕 最新功能：学习日程管理 (2025-12-29)

### Feature: 学习日程管理 🔄

**状态**: 93% 完成 - API测试通过，待前端测试  
**完成时间**: 2025-12-29（实现完成 + API测试完成）

#### 功能概述
学习日程（Study Schedule）是一个全新的时间管理功能，与学习计划（Study Plan）互补：
- **学习计划** = 目标管理（例如：本月要做100道题）
- **学习日程** = 时间安排（例如：今天9:00-10:00背单词）

#### 核心功能

**1. 精确时间管理**
- 精确到分钟的时间安排
- 支持跨天日程
- 自动计算时长
- 时间冲突检测

**2. 8种活动类型**
- 背单词/记忆、听课/看视频、做题/练习、复习笔记
- 模拟考试、阅读教材、写作练习、休息

**3. 13种科目支持**
- 公共课：政治、英语、数学
- 计算机专业课：数据结构、计算机组成原理、操作系统、计算机网络
- 其他专业课：经济学、管理学、法学、专业课一、专业课二

**4. 重复日程**
- 单次/每天/每周重复
- 自动创建重复日程实例

**5. 多维度统计**
- 按活动类型、科目统计
- 完成率、学习时长统计

```python
class NoteService:
    QUESTION_LINK_PATTERN = re.compile(r'\[\[Q:(\d+)\]\]')
    
    @staticmethod
    def extract_linked_questions(content: str) -> List[int]:
        """从笔记内容中提取链接的题目ID"""
        matches = NoteService.QUESTION_LINK_PATTERN.findall(content)
        return [int(qid) for qid in matches]
    
    @staticmethod
    def create_note(user_id: int, note_data: dict) -> QuestionNote:
        """创建笔记，自动提取题目链接"""
        linked_questions = NoteService.extract_linked_questions(content)
        note = QuestionNote(
            content=content,
            tags=note_data.get('tags', []),
            linked_questions=linked_questions
        )
```

#### 关键变更说明

**从当前实现需要改变的内容**：

1. **链接语法**：
   - **当前**：`[[Q:123]]`（使用题目ID）
   - **新**：`[[题:题目标题]]`（使用题目标题）
   - **原因**：用户完成题目后不记得ID

2. **链接存储**：
   - **当前**：`linked_questions: [123, 456]`（仅ID）
   - **新**：`linked_questions: [{"question_id": 123, "title": "...", "link_text": "..."}]`
   - **原因**：需要存储ID和标题以正确渲染

3. **题目搜索**：
   - **当前**：按ID或关键词搜索，显示内容
   - **新**：按标题/内容搜索，突出显示标题
   - **原因**：用户需要看到题目标题来选择正确的题目

4. **链接插入**：
   - **当前**：`insertQuestionLink()` 插入 `[[Q:${question.id}]]`
   - **新**：`insertQuestionLink()` 插入 `[[题:${question.title}]]`
   - **原因**：在链接文本中使用标题而不是ID

5. **链接渲染**：
   - **当前**：显示"题目 #123"
   - **新**：显示题目标题文本
   - **原因**：对用户更有意义的显示

#### 10个正确性属性

设计文档包含10个正确性属性，用于基于属性的测试：

1. **链接提取完整性**：提取的链接数量应等于内容中的链接模式数量
2. **链接插入幂等性**：在同一位置插入两次应产生两个独立的链接实例
3. **Markdown 渲染安全性**：渲染输出不应包含可执行脚本
4. **搜索防抖正确性**：300ms内的多次输入应只触发一次搜索
5. **题目标题解析**：有效的链接文本应解析为正确的题目ID和标题
6. **内容长度验证**：验证应拒绝<1或>5000字符的内容
7. **链接格式向后兼容性**：旧格式链接应正确渲染并在更新时转换
8. **光标位置保留**：格式化操作后光标应在正确位置
9. **预览同步**：预览应在100ms内反映内容变化
10. **链接唯一性**：同一题目的多个链接应在存储中只有一个条目
