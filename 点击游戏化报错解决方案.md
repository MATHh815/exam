# 点击"游戏化"菜单报错解决方案

## 错误现象
点击个人中心的"游戏化"菜单时，出现：
```
基于了未知的路径格式
```

---

## 原因
后端服务器没有运行，或者用户积分记录未初始化。

---

## 解决方案

### 方案 1: 一键启动（推荐）⭐

1. 打开命令行（CMD）
2. 进入 exam 目录：
   ```bash
   cd C:\Users\ABEL\Desktop\ABS\exam
   ```
3. 运行启动脚本：
   ```bash
   start_all.bat
   ```
4. 等待 10 秒
5. 刷新浏览器页面（按 F5）
6. 重新点击"游戏化"菜单

**完成！** ✓

---

### 方案 2: 手动启动

#### 第一步：启动后端服务器

1. 打开命令行（CMD）
2. 进入后端目录：
   ```bash
   cd C:\Users\ABEL\Desktop\ABS\exam\backend
   ```
3. 启动服务器：
   ```bash
   python run.py
   ```
4. 看到这个提示就成功了：
   ```
   * Running on http://127.0.0.1:5000
   ```
5. **保持这个窗口打开！不要关闭！**

#### 第二步：初始化用户积分

1. **新开一个命令行窗口**（不要关闭第一个）
2. 进入后端目录：
   ```bash
   cd C:\Users\ABEL\Desktop\ABS\exam\backend
   ```
3. 运行初始化脚本：
   ```bash
   python init_user_points.py
   ```
4. 看到"初始化完成！"就成功了

#### 第三步：刷新浏览器

1. 回到浏览器
2. 按 `F5` 刷新页面
3. 重新点击"游戏化"菜单

**完成！** ✓

## 常见问题

### Q: 运行 start_all.bat 后没反应？

**答**: 等待 10-15 秒，后端和前端需要启动时间。

### Q: 提示"端口被占用"？

**答**: 
1. 关闭所有之前打开的命令行窗口
2. 重新运行 start_all.bat

### Q: 提示"No module named 'flask'"？

**答**: 
```bash
cd exam\backend
pip install -r requirements.txt
```

### Q: 刷新后还是报错？

**答**: 
1. 确认后端服务器窗口还在运行
2. 访问 http://localhost:5000/health 检查后端
3. 如果无法访问，重新启动后端

### Q: 积分显示 0 是正常的吗？

**答**: 是的！新用户初始积分为 0。

**获取积分的方法**：
- 完成练习题
- 完成模拟考试
- 完成每日任务
- 解锁成就
